Definitions

git add = add file changes to the staging area, just before commit

git commit = commit the changes that has been staged, this will create a new stamp/history in the version control

git branch = list all local branches

git switch = to switch to a different branch

git log --oneline --graph --all = git log shows commit history, oneline means each commit is represented as one line, graph means graphical representation

branch = within the same repo, but a separate pipeline/flow that starts out the same as the main repo at creation but can be version controlled separately, useful in a team setting, then can be merged with the main pipeline down the road

pull request = a request to merge a branch with main, to proceed a review and approval is often required to ensure changes are appropriate
